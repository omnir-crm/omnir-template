## Getting Started

### For Developers

1. **Setup Environment**
   ```bash
   docker-compose up -d --build
   ```

2. **Access Application**
   - Web UI: http://localhost:8080
   - Database: localhost:3306

3. **Read the Guides**
   - Start with [Development Guide](./development-guide.md)
   - Reference [Architecture](./architecture.md) for system understanding

### For API Consumers

1. Read [API Contracts](./api-contracts.md) for available endpoints
2. Authenticate using the challenge-response flow
3. Use VTQL for complex queries

### For Database Work

1. Review [Data Models](./data-models.md) for schema understanding
2. Check entity relationships before creating queries
3. Use `vtiger_crmentity` as the base for all CRM records

## AI-Assisted Development

When working with AI assistants on this codebase:

1. **Context Files**
   - Primary: [CLAUDE.md](../CLAUDE.md)
   - Architecture: [architecture.md](./architecture.md)
   - Structure: [source-tree-analysis.md](./source-tree-analysis.md)

2. **Key Patterns to Understand**
   - Class naming: `{Module}_{Component}_{Type}`
   - File mapping: `Accounts_Detail_View` → `modules/Accounts/views/Detail.php`
   - Inheritance: Module → Parent → Vtiger (fallback)

3. **When Adding Features**
   - Check existing module implementations first
   - Follow the MVC pattern in `modules/{Module}/`
   - Use the Vtiger_Loader for autoloading
